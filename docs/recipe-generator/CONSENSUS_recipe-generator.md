# 基于大模型的个性化食谱生成系统 - 共识文档

## 1. 明确的需求描述和验收标准

### 1.1 功能需求
- **食材管理模块**：支持文本与图像双模态食材输入，实现食材自动分类、存储与智能替代推荐
- **用户画像模块**：收集用户基础信息、健康目标、饮食限制、口味倾向与烹饪熟练度，支持画像实时更新
- **食谱生成模块**：根据用户现有食材与画像生成3-5个候选食谱，按烹饪熟练度调整步骤详细度，支持食谱收藏功能
- **营养分析模块**：计算28项核心营养指标，生成符合《中国居民膳食指南》的评估报告，提供可落地的营养调整建议
- **系统管理模块**：实现用户管理、食材营养数据库更新、操作日志管理等管理员功能

### 1.2 非功能需求

### 4.2 技术实现细节
- **食材识别流程**：用户上传食材（文本/图像）→ 文本食材通过BERT模型解析名称与数量 → 图像食材通过Agent模型识别 → 多模态信息协同校验 → 食材自动分类 → 存储至数据库 → 若食材不足则查询替代知识库推荐替代方案
- **食谱生成流程**：接收用户食材与画像数据 → 构建结构化提示词 → 调用ChatGLM-4模型 → 生成初始食谱 → 通过食材搭配规则库校验 → 过滤不合理食谱 → 按烹饪熟练度调整步骤详细度 → 生成3-5个候选食谱 → 支持收藏功能
- **营养分析流程**：接收食谱食材构成数据 → 查询食材营养数据库 → 计算食谱总营养成分 → 对照用户健康目标与《中国居民膳食指南》→ 生成营养评估报告 → 基于食材替代方案提供调整建议
