# 基于大模型的个性化食谱生成系统 - 对齐文档

## 1. 项目上下文分析

### 1.1 项目结构与技术栈
- **前端技术**：Vue.js 3、Element Plus、Axios
- **后端技术**：Spring Boot 3、MyBatis-Plus、Spring Security
- **AI模型**：ChatGLM-4（食谱生成）、Agent模型（食材图像识别）、BERT（文本食材解析）
- **数据存储**：MySQL 8.0（结构化数据）、Redis 6.0（缓存）
- **其他工具**：Git、JMeter、Postman、Maven

### 1.2 架构模式
- 采用B/S架构结合分层架构设计
- 融入微服务思想实现核心功能解耦
- 系统整体分为表现层、业务逻辑层、数据访问层、数据存储层

### 1.3 业务域与数据模型
- **业务域**：个性化食谱生成、多模态食材识别、营养分析
- **核心数据模型**：用户、用户偏好、食材、食谱、营养数据

## 2. 需求理解确认

### 2.1 原始需求
- 基于大模型的个性化食谱生成系统
- 支持多方式且精准的食材输入（文本与图像识别）
- 多维度个性化食谱推荐（结合健康目标、口味偏好、烹饪熟练度）
- 精准且实用的营养分析（含成分计算与调整建议）
- 简洁易用的操作界面

### 2.2 边界确认
- **功能边界**：五大核心模块（食材管理、用户画像、食谱生成、营养分析、系统管理）
- **技术边界**：使用指定的技术栈和AI模型
- **性能边界**：页面响应时间≤3秒、食材识别准确率≥90%
- **安全边界**：API密钥使用.env文件管理，确保系统安全

### 2.3 需求理解
- **食材管理模块**：支持文本与图像双模态食材输入，实现食材自动分类、存储与智能替代推荐
- **用户画像模块**：收集用户基础信息、健康目标、饮食限制、口味倾向与烹饪熟练度，支持画像实时更新
- **食谱生成模块**：根据用户现有食材与画像生成3-5个候选食谱，按烹饪熟练度调整步骤详细度，支持食谱收藏功能
- **营养分析模块**：计算28项核心营养指标，生成符合《中国居民膳食指南》的评估报告，提供可落地的营养调整建议
- **系统管理模块**：实现用户管理、食材营养数据库更新、操作日志管理等管理员功能

### 2.4 疑问澄清
- **AI模型调用**：如何调用阿里云百炼平台的API？需要具体的调用方式和参数
- **食材识别**：图像识别的具体实现方式，如何处理低置信度的识别结果
- **食谱生成**：如何确保生成的食谱与用户食材匹配，如何处理食材不足的情况
- **营养分析**：如何计算不同烹饪方式对营养成分的影响
- **系统集成**：前后端如何集成，API接口的具体实现方式

### 3.1 歧义识别
- **AI模型调用**：需要明确阿里云百炼平台的API调用方式和参数
- **食材识别**：需要明确图像识别的具体实现方式和低置信度处理策略
- **食谱生成**：需要明确食材匹配逻辑和食材不足的处理策略
- **营养分析**：需要明确烹饪方式对营养成分的影响计算方法
- **系统集成**：需要明确前后端集成方式和API接口实现

### 3.2 决策依据
- 基于现有项目内容和行业知识，采用成熟的技术方案
- 参考Mealime、Yummly、下厨房等国内外主流产品的设计思路
- 结合用户调研结果，确保系统满足用户核心需求

### 3.3 关键决策点
- **AI模型选择**：使用ChatGLM-4作为食谱生成核心模型，Agent模型用于食材图像识别，BERT用于文本食材解析
- **架构设计**：采用分层架构设计，融入微服务思想实现核心功能解耦
- **数据存储**：使用MySQL存储结构化数据，Redis缓存热点数据
- **接口设计**：RESTful API设计，确保前后端通信顺畅
- **安全策略**：使用Spring Security保障系统安全，API密钥使用.env文件管理
